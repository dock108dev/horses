## Risk log: code intentionally retained

These would have been candidates to delete on a more aggressive pass but
were kept with concrete justification.

### `api/normalize.py` — kept

**Risk consideration**: `api/model.py` now contains a comment
("Duplicates the small bit of logic from
`api.normalize.normalize_probabilities` locally to avoid the circular
import"), which on its face suggests we should fold `normalize` into
`model`.

**Why kept**: `api/normalize.py` is the SSOT for *odds parsing*
(`odds_to_probability`, `_parse_odds_to_decimal`), *Equibase ⇄
TwinSpires merging* (`merge_horses`, `_match_by_name`,
`normalize_horse_name`), and *Pick 5 sequence-role assignment*
(`assign_pick5_sequence_roles`). Six callers (`api/refresh.py`,
`api/validate.py`, `api/sources/fixture.py`,
`api/tests/test_normalize.py`, `api/tests/test_refresh.py`,
`api/tests/test_main.py`) read it. Folding it into `api/model.py`
would push that file past 1700 LOC and force a circular
`model ⇄ normalize` graph through the merge functions. The duplication
inside `_renormalize` is intentional and called out in-place.

### `api/refresh.py` — kept

`api/main.py` could absorb `build_card` / `poll_pick5_odds` /
`races_with_latest_odds` directly. Kept as a separate module because:
it's the one place all three source adapters meet, it's pure-functional
(no FastAPI types), and its tests don't need the `app`/`Envelope`
fixtures.

### `web/components/{SimulationSummary,TicketBuilder}.tsx` — duplicate `pct` / `num` / `money` formatters retained (superseded)

Documented in `docs/audits/cleanup-report.md` "Things deliberately not
touched": the `digits` defaults differ between files (1 vs 2 in `pct`),
so a no-behavioral-change extraction is impossible without changing
either every call-site or a default. The follow-up cleanup pass acted
on this — `digits` was made required at every call site and the
formatters were extracted to `web/lib/format.ts`. See
`docs/audits/cleanup-report.md` "Duplicates consolidated".

### Inline references to `docs/audits/cleanup-report.md` and `docs/audits/error-handling-report.md`

These files exist on disk, so pointers to them are valid. Several inline
comments cite them; left intact.
